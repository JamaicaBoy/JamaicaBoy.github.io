注塑论文知识库 RAG Agent（Agentic RAG + LangGraph）Injection-molding Paper Knowledge-base RAG Agent (Agentic RAG + LangGraph)

学生排一 · 独立实现First student author · built independently 中文核心《精密成形工程》· 已发表CSCD-core (J. Netshape Forming Eng.) · published Agentic RAG · LangGraphAgentic RAG · LangGraph GitHub 开源 DemoOpen-source demo on GitHub

面向注塑成型企业的 RAG + Agent 论文知识库应用：以 819→896 篇中英文注塑工艺论文为核心知识来源，完成 PDF 解析、章节清洗、元数据整理、知识卡抽取与 section-aware chunk 切分，构建向量知识库与 Agentic RAG 检索问答流程；支持工艺问答、缺陷诊断、工艺参数影响分析、方法对比、证据追溯和多轮对话记忆，并通过 LangChain 适配层与 LangGraph 显式状态图升级为可扩展的企业级 Agent Workflow。离线评测端到端任务完成率 86%，证据引用准确率 91%，无证据问题拒答率 95%；并在该知识库基础上扩展出工艺参数推荐应用，3 轮内收敛至目标质量。An RAG + Agent paper knowledge-base application for injection-molding manufacturers: 819→896 Chinese and English molding-process papers are parsed, cleaned, enriched with metadata, distilled into knowledge cards, and split into section-aware chunks to build a vector knowledge base and an Agentic RAG retrieval-QA pipeline. It supports process Q&A, defect diagnosis, parameter-impact analysis, method comparison, evidence tracing and multi-turn conversation memory, and has been upgraded via a LangChain adapter layer and an explicit LangGraph state graph into an extensible enterprise-grade Agent Workflow. Offline evaluation shows an 86% end-to-end task completion rate, 91% evidence-citation accuracy and a 95% refusal rate on evidence-free questions; a process-parameter recommendation feature built on top of this knowledge base converges to the target quality within 3 rounds.

背景Background

  • 论文知识分散，检索成本高：注塑成型论文涉及材料、模具、工艺参数、缺陷机理、质量预测、优化算法、仿真建模等多个方向，仅靠人工翻 PDF 很难在短时间内找到可靠证据。Scattered paper knowledge, high retrieval cost: molding papers span materials, molds, process parameters, defect mechanisms, quality prediction, optimization algorithms and simulation modeling — manually skimming PDFs rarely surfaces reliable evidence quickly.
  • 缺陷诊断依赖经验，知识沉淀不足：翘曲、缩水、熔接痕、短射、飞边、气泡、烧焦、透明件发雾等缺陷通常与熔体温度、模具温度、注射速度、保压压力、冷却时间等参数相关，但不同材料、模具结构和工况下论文结论可能存在差异，需要结合证据判断。Defect diagnosis relies on experience, with little knowledge retained: warping, sink marks, weld lines, short shots, flash, bubbles, burning and haze on clear parts usually relate to melt temperature, mold temperature, injection speed, holding pressure and cooling time — but conclusions can differ across materials, mold structures and operating conditions, so evidence-based judgment is needed.
  • 工艺参数影响关系复杂：单个参数可能同时影响重量、尺寸精度、透过率、表面质量、收缩率和内部缺陷，企业需要的不只是"参数升高会怎样"，而是"在哪些论文、哪些材料、哪些质量指标下有证据"。Complex parameter-quality relationships: a single parameter can affect weight, dimensional accuracy, transmittance, surface quality, shrinkage and internal defects at once — what's needed isn't just "what happens if this goes up" but "which papers, which materials, which indicators have evidence for it."
  • 工业场景需要可追溯、可复核：生产参数调整、质量放行、设备异常、安全风险等问题不能让大模型直接拍脑袋回答，系统需要提供证据来源、置信度判断和人工复核入口。Industrial scenarios need traceability and review: production parameter changes, quality sign-off, equipment anomalies and safety risks can't be answered by an LLM off the cuff — the system must surface evidence sources, confidence, and a human-review path.

方法 / 系统架构Method / Architecture

  • 知识库构建：819→896 篇中英文论文 → PDF 解析（正文/标题/表格/图题/公式附近文本）→ 清洗页眉页脚与参考文献噪声 → 按论文结构做 section-aware chunk → 生成 metadata、论文卡、缺陷卡、参数卡 → embedding 写入 Chroma 向量库，同步构建 BM25 索引。Knowledge-base construction: 819→896 papers → PDF parsing (body text, titles, tables, figure captions, formula context) → cleaning headers/footers and reference noise → section-aware chunking (not fixed-length) → metadata, paper/defect/parameter cards → embeddings into a Chroma vector store, plus a parallel BM25 index.
  • Agentic RAG 检索问答：BM25（精确术语召回）+ Dense Retrieval（语义召回）+ Rerank 精排，输出带来源、相似度与 rerank 分数的 Evidence Table；设计 Query Rewrite 对口语化问题做术语归一、意图识别和中英文扩展。Agentic RAG retrieval-QA: BM25 (exact-term recall) + dense retrieval (semantic recall) + reranking produce an Evidence Table with sources, similarity and rerank scores; Query Rewrite normalizes terminology, recognizes intent, and expands colloquial questions across Chinese/English.
  • Agent 工具链（LangGraph 状态图编排）：search_papers_tool（论文证据检索）、defect_diagnosis_tool（缺陷诊断）、parameter_effect_tool（工艺参数影响分析）、method_compare_tool（论文方法对比）、evidence_extract_tool（证据表整理）、human_review_tool（人工复核）、knowledge_gap_tool（知识缺口记录）。Agent tool chain (orchestrated as a LangGraph state graph): search_papers_tool (evidence retrieval), defect_diagnosis_tool (defect diagnosis), parameter_effect_tool (parameter-impact analysis), method_compare_tool (method comparison), evidence_extract_tool (evidence-table assembly), human_review_tool (human review), and knowledge_gap_tool (knowledge-gap logging).
  • Memory 与长对话：短期对话记忆保存最近轮次与关键实体，支持"它/刚才那种缺陷"等指代消解；长对话自动压缩为结构化摘要（用户目标、关键实体、已确认结论、待解决问题、风险等级），降低上下文压力，但结论始终以当前检索证据为准。Memory & long-context handling: short-term memory keeps recent turns and key entities to resolve references like "it" or "that defect"; long conversations are compressed into a structured summary (goal, entities, confirmed findings, open questions, risk level) to reduce context pressure, while conclusions always defer to the current retrieval evidence.
  • 多 Corpus Mode 与增量索引：dev（约 30 篇，开发调试）/ full（本地 896 篇，效果最完整）/ public_full_release（公开 Demo 默认，通过 GitHub Release 分发结构化产物，不含 PDF 原文，避免仓库过大）；用户上传 PDF 实时解析、chunk、embedding 后写入独立 collection，仅作增量补充，不污染主知识库。Multiple corpus modes & incremental indexing: dev (~30 papers, for development) / full (896 papers locally, best fidelity) / public_full_release (default for the public demo, distributed as structured artifacts via GitHub Release, excluding raw PDFs to keep the repo small); user-uploaded PDFs are parsed, chunked and embedded on the fly into a separate collection as incremental supplements only.

结果与亮点Results & Highlights

  • 离线评测：端到端任务完成率 86%，证据引用准确率 91%，无证据问题拒答率 95%。Offline evaluation: 86% end-to-end task completion, 91% evidence-citation accuracy, 95% refusal rate on evidence-free questions.
  • 覆盖翘曲、缩水、熔接痕、短射、飞边、气泡、烧焦、透明件发雾等缺陷诊断，以及熔体温度、模具温度、注射速度、保压压力、冷却时间等参数影响分析。Covers diagnosis of warping, sink marks, weld lines, short shots, flash, bubbles, burning and haze, plus impact analysis of melt temperature, mold temperature, injection speed, holding pressure and cooling time.
  • 已通过 GitHub Release 发布 full_release_no_pdf_v1 数据包（不含 PDF 原文，公开可复现）。Published the full_release_no_pdf_v1 data package via GitHub Release (no raw PDFs, publicly reproducible).
  • 在该知识库 Agent 基础上扩展出工艺参数推荐应用：PC、PMMA 等多材料仿真优化任务均可 3 轮内收敛至目标质量范围；相关内容以学生排一发表中文核心论文 1 篇（《精密成形工程》），国家发明专利 1 项实质审查中（学生排一）。A process-parameter recommendation application built on this knowledge-base agent converges to the target quality within 3 rounds across multi-material simulation tasks (PC, PMMA, etc.); related work includes one Chinese core journal paper published as first student author (Journal of Netshape Forming Engineering) and one national invention patent under substantive examination (first student contributor).
